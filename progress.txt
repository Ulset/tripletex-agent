## Codebase Patterns
- PlanExecutor resolves `$stepN.path.to.value` placeholders via `_resolve_placeholders()` — works recursively on dicts, lists, and strings
- `_traverse()` supports dot-notation traversal into nested dicts and lists (list indices as integers)
- Use `.venv/bin/pytest` to run tests (python -m pytest may fail due to path issues)
- Python 3.12 required - use `python3.12 -m venv .venv` to create virtualenv
- Config uses dataclass with `field(default_factory=lambda: os.getenv(...))` pattern
- FastAPI app in src/main.py, config in src/config.py
- Dev dependencies installed via `pip install -e ".[dev]"`
- TripletexClient uses Basic Auth with username "0" and session_token as password
- Use `responses` library to mock HTTP calls in TripletexClient tests
- `responses.calls[0].request.body` returns bytes, not str — use `b"..."` for assertions
- FileProcessor uses pymupdf (fitz) for PDF text extraction, OpenAI vision API as fallback for scanned PDFs/images
- Mock OpenAI with `unittest.mock.patch("src.file_processor.OpenAI")` at class level
- Mock OpenAI in plan_generator with `@patch("src.plan_generator.OpenAI")` — construct MagicMock return chain for chat.completions.create
- TaskOrchestrator in orchestrator.py: mock at `@patch("src.main.TaskOrchestrator")` for endpoint tests; for orchestrator tests, mock both `src.plan_generator.OpenAI` and `src.file_processor.OpenAI` plus use `responses` for Tripletex
- Structured JSON logging via `src/logging_config.py` — `setup_logging()` configures root logger with JSONFormatter and SensitiveFilter
- SensitiveFilter redacts values matching session_token, api_key, password, secret, authorization, bearer patterns in log messages and args
- ExecutionResult tracks `total_api_calls` and `error_count`; PlanExecutor accumulates these across replans via `execute_with_replan`
- PlanExecutor.replan_count is set during `execute_with_replan` — read by orchestrator for efficiency logging
- E2E tests in `tests/e2e/` — run with `.venv/bin/pytest -m e2e`, require TRIPLETEX_BASE_URL + TRIPLETEX_SESSION_TOKEN env vars (or .env.test)
- E2E conftest provides `tripletex_client`, `sandbox_credentials`, and `openai_api_key` fixtures with auto-skip on missing credentials
